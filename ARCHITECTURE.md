## Key Components

### 1. Autopager Position (CORRECTED)
**Location:** After "Extract HTML & DOM" and BEFORE branching into Structural/Behavioral paths

**Purpose:** 
- Primary detection mechanism
- Extracts pagination links from HTML
- Provides foundation for both paths

### 2. Detection Stack Priority

**Structural Path (Has Pagination):**
1. âœ… **Autopager** (Primary)
2. XPath/CSS Selectors (Backup)
3. Pattern Matching (Deep scan)
4. Platform Specific (Edge cases)

**Behavioral Path (No Pagination):**
1. Scroll Testing (First attempt)
2. Button Detection (Second attempt)
3. Click Verification (Confirmation)
4. Iframe Checks (Recursive)

### 3. AI Judge Usage

**Triggered When:**
- Heuristics are unclear
- Timeout/Error occurred
- Conflicting signals detected
- Manual override needed

**Not Used When:**
- Clear detection from autopager
- Strong heuristic signals
- No API key provided

- **print_lock**: Console output synchronization
- **file_lock**: CSV write synchronization
- **task_queue**: Worker coordination

## Performance Optimization

- Parallel workers (configurable)
- Headless mode (faster)
- Incremental saving (no data loss)
- Timeout recovery (continue on errors)
- Rate limiting for AI calls (5/min)
